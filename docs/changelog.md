
# سجل تطوير مشروع NovaFlow ERP 🚀

## [1.3.9] - بناء محرك المسارات الفنية (Technical Path Engine)
### ✅ التحديثات (Updates)
- **توحيد المسارات:** اعتماد مسمى `technicalStages` كمرجع وحيد للـ WBS وإلغاء التضارب التاريخي مع `workStages`.
- **هيكلة مرجعية ثلاثية:** إعادة تنظيم القوائم المرجعية إلى (تنظيمية، جغرافية، وفنية) وفقاً للدستور البرمجي الجديد.
- **واجهة Drill-down:** بناء واجهة متطورة تسمح بالتنقل الشجري بين (نوع المعاملة -> الخدمة الفرعية -> مراحل العمل).
- **أتمتة SaaS:** ربط كافة المرجعيات بـ `companyId` لضمان استقلالية كل منشأة في بناء محركها التشغيلي الخاص.
- **الامتثال للـ UI Policy:** تطبيق سياسة الحقول النظيفة (لا قيم افتراضية، لا أسهم) في كافة نوافذ الإدخال المرجعية.
