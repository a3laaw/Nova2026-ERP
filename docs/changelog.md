
# سجل تطوير مشروع NovaFlow ERP 🚀

## [1.3.0] - نظام التفعيل الذاتي (Self-Service SaaS)
### ✅ التحديثات (Updates)
- **التفعيل التلقائي:** تحويل صفحة التسجيل `/register` لتقوم بإنشاء المنشأة (Company) وحساب المستخدم (Global User) فوراً دون الحاجة لموافقة المطور.
- **إدارة الهوية:** إضافة حقول "اسم المستخدم" و "كلمة المرور" في نموذج التسجيل لتمكين الدخول الفوري.
- **سياسة التفعيل الآلي:** تطبيق قاعدة (14 يوماً تجريبية + 5 مستخدمين) برمجياً عند إنشاء السجل.
- **تحسين تجربة المستخدم (UX):** إضافة ميزة "إظهار/إخفاء كلمة المرور" وتوجيه المستخدم تلقائياً للوحة التحكم بعد النجاح.
- **تحديث البنية التحتية:** استخدام `writeBatch` لضمان تكامل البيانات بين ملفات الشركة والمستخدم العالمي في عملية واحدة.

## [1.2.9] - أتمتة تفعيل المنشآت (SaaS Provisioning)
- **محرك التجهيز:** إضافة منطق إنشاء مستند الشركة تلقائياً عند طلب الانضمام.
- **قيود الاشتراك:** برمجة تاريخ انتهاء تلقائي (بعد 14 يوماً) وحد أقصى للمستخدمين (5).
- **لوحة المطور:** تحديث لوحة المطور لتعمل كـ Audit Log للشركات المفعلة تلقائياً.

## [1.2.8] - نظام SaaS والتحكم في المنشآت
- **تحويل الـ CRM:** ربط صفحة العملاء بقاعدة بيانات حية معزولة لكل شركة (Multi-tenancy).
- **إدارة المسارات:** إنشاء ملف `multi-tenant.ts` لتوحيد الوصول لمجموعات البيانات الفرعية للشركات.
- **الأمن البرمجي:** تحديث قواعد الحماية لمنع تداخل بيانات الشركات المختلفة.

## [1.2.7] - دعم ثنائي اللغة (Bilingual Support)
- **التعريب الكامل:** إضافة LanguageContext وإضافة الترجمات للغتين العربية والإنجليزية.
- **واجهة مستخدم مرنة:** دعم اتجاهات RTL و LTR تلقائياً بناءً على اللغة المختارة.
- **زر التبديل:** إضافة زر تبديل اللغة في الهيدر لسهولة الوصول.

## [1.2.6] - التأسيس المعماري (Core Architecture)
- **بوابة الدخول الموحدة:** دمج دخول المطور والمستخدمين في صفحة واحدة مع توجيه ذكي.
- **لوحة المطور:** إنشاء واجهة خاصة للمطور (`/developer`) لإدارة النظام.
- **نموذج الطلبات:** إعداد `company_requests` لاستقبال طلبات الانضمام الجديدة.

## [1.0.0] - إطلاق المشروع
- البدء في بناء الهيكل الأساسي لـ NovaFlow ERP.
- ربط خدمات Firebase (Auth, Firestore).
